What does 64-bit mean?

Most computers these days are 64-bit, but what does that mean? Well let’s look at a 64-bit address:

0xdeadbeef12341234

Now the length of the address is 8 bytes. But I counted 16? Well that is not bytes, remember that 0x41 is the byte value for A. Bytes can hold 255 different values, and therefor needs 2 integers to represent it.

Let’s look at 0x41, as our example. 0x41 consists of two numbers, these numbers are each represented in binary, and are represented individually. Let’s look at 1.

0000 0001

Let’s then look at 4:

0000 0100

Combined together that leaves us with:

0100 0001

Which is the same as 0x41, which is the same as “A”

Essentially a byte is made of a lower and upper nibble. These can max have: 1111, otherwise it would overflow, and not be 4 bits of size. This means that each byte is 8 bits. If an address has 8 bytes which each is 8 bits, that would be 8*8 = 64. This is why it’s called a 64-bit system.

Signedness

When dealing with integers in computer science, there’s a lot of different ways of representing them - however it’s always important to know whether you’re dealing with an unsigned integer or a signed integer.

Consider the sign “-”. If a number is: -10, -20, -30, -x … then it is considered a signed integer. However if it doesn’t have any prefixed sign, then it would be considered an unsigned integer.

So this would make the largest unsigned integer value in a 32-bit[4 bytes]:

1111 1111 1111 1111 1111 1111 1111 1111

or 

FFFF FFFF

or 

4294967295

The largest unsigned integer value in a 64-bit[8 bytes] would be double:

1111 1111 1111 1111 1111 1111 1111 1111 1111 1111 1111 1111 1111 1111 1111 1111

or 

FFFF FFFF FFFF FFFF

or 

18446744073709551615

So this is the largest values for an unsigned integer, meaning that we don’t consider the -1, -5, -7 representatives. Now if we would want to look at let’s say the signed integer max or range for 32-bit it would simply be half, as half the numbers would now be used for negative numbers:

Smart smart. Now let’s say we want to do calculate the following:

5-1

How would we go about doing that in hex? The smart people that programmers are decided that the -1 would be 0xfffffffff, the max value would be the -1, so that it’s easy to remember. Meaning that if we want to calculate the prior statement it would need to be
